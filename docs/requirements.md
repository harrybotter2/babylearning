## 1. プロダクト概要

### 1.1 プロダクトビジョン

乳幼児向けドッツカードメソッドを、最新の認知科学研究でアップデートした、科学的根拠に基づくドッツアプリ。178 日間のプログラムが全自動で進行し、毎日ワンタップ・1 分で完了。科学的な裏付けを親に伝えながら、挫折せずに完走できるドッツアプリ。

### 1.2 コンセプト

| 項目 | 内容 |
|------|------|
| アプリ名 | Science Dots（サイエンス・ドッツ） |
| コンセプト | 認知科学 × ドッツカードメソッド |
| プラットフォーム | iOS / Android（Expo） |
| 販売モデル | 買い切り ¥1,480 |
| データ通信 | 不要（完全ローカル動作） |
| 対象年齢 | 生後 3 ヶ月〜3 歳（親が使用） |
| 初期マーケット | 日本 |
| 言語 | v1: 日本語 / v2: 英語対応 |

### 1.3 キータグライン

> "科学的根拠に基づくドッツメソッドで、赤ちゃんの数の感覚を育てる"

## 3. 差別化戦略

### 3.1 差別化軸①：科学的エビデンス

| 従来の主張 | 科学的評価 | アプリでの反映 |
|-----------|-----------|---------------|
| 乳児は数の量を直感的に認識できる | ○ 支持（ANS 研究） | 量の認識ステップを核に採用 |
| 大きな数も正確に区別できる | × 否定（Weber の法則） | 難易度を比率依存に設計（1:2→3:4→5:6） |
| 赤ドット固定が最適 | △ 一部否定 | 多様な表象（色・形・配置変化）で抽象化を促進 |
| 178 日固定カリキュラム | △ 個人差未考慮 | 月齢別コース（速度・サイズ最適化） |
| 短時間・高頻度セッション | ○ 支持 | そのまま採用（1 回 40 秒以内） |
| テスト禁止・楽しさ最優先 | ○ 支持 | テスト機能なし。親向けガイダンスのみ |

### 3.2 サイエンスカード（親向け知識）

カリキュラムの節目に「サイエンスカード」（1 画面・30 秒で読める豆知識）を挿入。初回 MVP では 30 枚程度。

| タイミング | サイエンスカード例 |
|-----------|-----------------|
| Day 1 | 赤ちゃんは生まれながらに"数の感覚"を持っています（Izard et al., 2009） |
| Day 7 | なぜ大きな数から始めるの？ — Weber の法則の解説 |
| Day 14 | ウェーバーの法則：8 と 16 の違いはわかっても、8 と 9 の違いは難しい |
| Day 30 | ANS（近似的数量感覚）の精度と将来の数学成績の相関（Halberda et al., 2008） |
| Day 50 | 複数の形で同じ数を見せることで、"5 はいつも 5"という抽象的理解が育ちます |
| Day 80 | 足し算の概念は"合体"を見せるだけ。赤ちゃんの脳は自然に理解します |
| Day 120 | スビタイジング（瞬時に数を把握する能力）は大人より幼児の方が優れています |
| Day 150 | 数字（記号）を導入するベストタイミングは、量の理解が十分育った後です |

### 3.3 差別化軸②：挫折防止設計

| 挫折原因 | 解決策 | 実装方法 |
|---------|--------|---------|
| 準備が面倒 | アプリ化で解決済み | ワンタップで今日のレッスン開始 |
| 何を見せるかわからない | 自動カリキュラム管理 | 178 日分のプログラムをローカル DB で管理 |
| 効果が見えない | 進捗ダッシュボード | カレンダー + ストリーク + マイルストーン |
| 完璧主義で疲弊 | お休みしても OK 設計 | スキップしても罰なし。再開メッセージで歓迎 |
| いつやるか忘れる | ローカル通知 | 毎日指定時刻にリマインド（「たった 40 秒！」） |

## 5. 画面設計

### 5.1 画面一覧

| # | 画面名 | 説明 |
|---|--------|------|
| S1 | オンボーディング | 初回起動時のみ。3 ステップのコンセプト説明 |
| S2 | コース選択 | 月齢別コースを選択（0-6M / 6-18M / 18-36M） |
| S3 | ホーム | 今日のレッスンボタン + 進捗サマリー + ストリーク |
| S4 | レッスン再生 | ドッツフラッシュ画面。全画面表示、自動進行 |
| S5 | サイエンスカード | レッスン後に表示される科学豆知識 |
| S6 | レッスン完了 | 完了アニメーション + 「明日も楽しみに！」 |
| S7 | 進捗ダッシュボード | カレンダー + 到達範囲 + 統計 |
| S8 | 設定 | 通知時刻 / フラッシュ速度 / 音声 ON・OFF / コース変更 |

### 5.2 画面遷移フロー

```
オンボーディング(S1) → コース選択(S2) → ホーム(S3)

ホーム(S3) → [今日のレッスン]タップ
  → レッスン再生(S4)
  → サイエンスカード(S5) ※節目のみ
  → 完了(S6)
  → ホーム(S3)

ホーム(S3) → [進捗]タブ → 進捗ダッシュボード(S7)
ホーム(S3) → [設定]アイコン → 設定(S8)
```

### 5.3 デザイン原則

- **カラー**：ホワイトベース + アクセントブルー（科学的信頼感）+ 赤（ドットのみ）
- **フラッシュ画面**：白背景に赤ドット。最大限のコントラスト
- **ホーム画面**：大きな「今日のレッスン」ボタンが中央。迷わない UI
- **フォント**：Noto Sans JP（日本語）+ Inter（英数字）
- **アニメーション**：ドット表示時に微細な fade-in。過度な演出は不要

## 6. カリキュラム設計

### 6.1 月齢別コース設計

| コース | 対象 | ドットサイズ | フラッシュ速度 | 特徴 |
|--------|------|------------|--------------|------|
| ベビー | 0-6M | 大（28mm） | 1.5 秒/枚 | 視力未発達を考慮。高コントラスト |

| トドラー | 18-36M | 標準（15mm） | 0.7 秒/枚 | 等式モード中心。多様表象導入 |

### 6.2 178 日プログラム構成

| フェーズ | 期間 | 内容 | セッション/日 |
|---------|------|------|-------------|
| 量の認識 | Day 1-50 | 1〜100 のドットカード。比率依存の難易度設計 | 3 回×10 枚 |
| 等式導入 | Day 51-100 | 足し算・引き算をドットで表示 | 3 回×10 枚 + 3 等式 |
| 抽象化 | Day 101-140 | 多様な表象で同じ量を提示 | 3 回×10 枚 + 3 等式 |
| 深化 | Day 141-178 | 混合等式、大小比較 | 3 回×8 枚 + 5 等式 |

### 7.3 データモデル（SQLite）

| テーブル | カラム | 用途 |
|---------|--------|------|
| user_settings | course_id, flash_speed, sound_enabled, notification_time | ユーザー設定 |
| lesson_progress | day_number, completed_at, session_count | レッスン進捗 |

| milestones | type, achieved_at, day_number | マイルストーン達成 |

### 7.4 ドット配置アルゴリズム

毎回異なる配置を生成することで、子どもが「パターン」ではなく「量」を認識するよう促す。

1. キャンバス領域をグリッド分割（重なり防止）
2. 各ドットの中心座標をランダムに配置
3. ドット間の最小距離を保証（半径 × 1.2）
4. 数が大きい場合はドットサイズを自動縮小
